AI-Based Parametric Insurance for Gig Workers
📌 Problem
Gig workers on platforms like Zomato, Swiggy, and Amazon depend on daily income.
External factors like rain, heat, pollution, or curfews reduce working hours, causing 20–30% income loss.
There is no system that automatically compensates for this.

💡 Solution
An AI-based parametric insurance platform that:
Tracks real-world conditions
Detects income disruption
Triggers instant payouts (no claims needed)

👥 Users
Delivery partners
E-commerce agents
Daily-income gig workers

⚙️ Key Features
🤖 Risk-based premium (AI-driven)
💰 Weekly pricing (₹30 / ₹60 / ₹100)
⚡ Automatic payouts
🛡️ Built-in fraud detection

🧠 Model (Simple View)
Inputs: Weather + Location + Earnings + Activity
Outputs: Risk score + Premium + Payout decision

⚡ Trigger Logic
If:
Weather crosses threshold
Activity drops

➡️ Payout is triggered automatically
🚨 Anti-Spoofing Strategy
🔍 Real vs Fake Users
Real:
Natural movement
Active deliveries
Fake:
Static / repeated movement
No activity
Sudden location jumps

📊 Data Used (Beyond GPS)
Movement sensors (accelerometer)
Order activity
Network signals
Device fingerprint
Behavior patterns

🧩 Fraud Detection
Detects similar patterns across users
Identifies group attacks (fraud rings)
Uses clustering + anomaly detection

No instant rejection

OTP / activity verification

⚖️ Fairness
Genuine users not penalized
Re-check + manual review support
